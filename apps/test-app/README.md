# ToDo, a DappQL demo

A minimal Next.js app that exercises the core of [DappQL](https://dappql.com), a typed data layer for dApp frontends on top of wagmi + viem.

The app is a three-column ToDo board backed by a single Solidity contract on Sepolia. Every read, mutation, and refetch on screen is powered by DappQL hooks; there are no raw `useReadContract` / `useWriteContract` calls anywhere.

**Live:** deployed on Vercel. **Contract:** [`0x29B6…1E93F`](https://sepolia.etherscan.io/address/0x29B63f08aBa4Be48873238C23693a5550bC1E93F#code) on Sepolia.

## What this demo proves

| Feature | Where it shows up |
| --- | --- |
| Typed codegen from ABIs | `./src/contracts/` is generated by the [`dappql`](https://www.npmjs.com/package/dappql) CLI from `dapp.config.js`. The entire app imports `ToDo.call.*` / `ToDo.mutation.*` with full types. |
| Cross-component multicall batching | Three `List` components + the count query + the header block badge all fuse into one RPC per refetch cycle via `useContextQuery` and `useIteratorContextQuery`. |
| Per-block reactivity | The block badge in the header ticks with every Sepolia block; every read in the tree silently refetches at the same cadence (`watchBlocks` on the provider). |
| Iterator queries | The to-do list uses `useIteratorContextQuery(total, (i) => ToDo.call.item(account, i))`, one hook for a dynamic-length on-chain array. |
| Mutations with simulate / confirm | `useMutation` wraps add / update / status-change with typed args, preflight simulation, and receipt tracking. |
| Global transaction UX | `onMutationUpdate` on the provider drives every toast in the app; no per-component glue. |

Open the "Source" button in the header (the `< / >` icon) for the annotated DappQL code behind each feature.

## Run it locally

```bash
# From the monorepo root
pnpm install

# Then either:
pnpm --filter test-app dev
# or
cd apps/test-app && pnpm dev
```

Open [localhost:3000](http://localhost:3000). Connect an injected wallet (MetaMask, Rabby, Rainbow extension, Coinbase wallet extension) on Sepolia. You'll need a small amount of Sepolia ETH to sign mutations, grab some from [sepoliafaucet.com](https://sepoliafaucet.com) or similar.

Optional: set a custom RPC via `apps/test-app/.env.local`:

```
NEXT_PUBLIC_CHAIN_URL=https://sepolia.infura.io/v3/YOUR_KEY
```

Omit it to use wagmi's default public Sepolia RPC (rate-limited, fine for reading).

## How it's wired

**`src/components/DappProvider.tsx`** , sets up wagmi + react-query + DappQL. One injected connector, `watchBlocks` on, `simulateMutations` on, a single `onMutationUpdate` callback that fans out to toasts.

**`src/pages/index.tsx`** , the whole app in one file. The three columns share a single `useContextQuery` batch via the provider's query manager. Open the source drawer to see the annotated code.

**`dapp.config.js`** , points at the ToDo contract + its ABI. Run `npx dappql` from this directory to regenerate `./src/contracts/`.

## Contract

The ToDo contract is a simple five-method Solidity contract. Functions:

- `addItem(string content, uint256 status) → uint256` , returns the new item id.
- `updateItem(uint256 id, string content, uint256 status)`
- `updateStatus(uint256 id, uint256 status)` , move between Pending / Doing / Done.
- `numItems(address) → uint256` , per-account count.
- `item(address, uint256) → tuple` , per-account, by index.

Source and verified code: [Sepolia Etherscan](https://sepolia.etherscan.io/address/0x29B63f08aBa4Be48873238C23693a5550bC1E93F#code).

## Dependencies

```
@dappql/react      # hooks + provider
@dappql/async      # non-react bits (not used here but shipped for parity)
@chakra-ui/react   # UI kit
wagmi + viem       # chain + wallet
```

No WalletConnect, no RainbowKit, no backend. This is a single-file app that shows off what DappQL looks like end-to-end.
